# 도서 관리 시스템 기능 명세서

## 회원가입 및 로그인 기능

- **PocketBase Auth 기반의 사용자 인증**: 이메일 및 비밀번호를 기반으로 한 사용자 회원가입 및 로그인 기능 제공.
- **인증 연동 및 권한 관리**: 로그인한 사용자만 도서 등록(검색 등록/수동 등록), 대출 토글, 도서 삭제, 좋아요 등 데이터 생성/수정/삭제 권한 부여.
- **마이페이지 (`/me`) 제공**: 로그인 후 본인의 프로필 정보(이름, 이메일 등) 및 직접 등록(`user_id` 기준)한 도서 목록을 필터링하여 페이징 그리드로 조회 가능.
- **Hydration 오류 방지**: 클라이언트 브라우저 상태와 서버 사이드 렌더링 간의 불일치를 막기 위해 `isHydrated` 상태 체크 후 인증 종속 UI 렌더링 처리.

---

## 도서 목록 조회 및 정렬 기능

- **도서 목록 페이징 출력**: 시스템에 등록된 전체 도서를 8개 단위로 페이지네이션하여 그리드 뷰(`BookListView`)로 출력.
- **대시보드 시각화 (`DashboardChart`)**: Recharts의 `PieChart`를 활용하여 전체 도서 중 대출 가능(`isAvailable: true`) 도서와 대출 중(`isAvailable: false`) 도서의 비율을 실시간 차트로 시각화. 도서가 없을 경우 차트 렌더링 생략.
- **조건별 도서 정렬**: 드롭다운 필터를 통해 최신순(`-created`), 오래된 순(`created`), 제목순(`title`) 정렬 지원.
- **상세 정보 뷰 스위칭**: 목록에서 도서 카드 클릭 시 페이지 전환 없이 부모 컴포넌트의 상태 관리를 통해 상세 정보 뷰(`BookDetailView`)로 부드럽게 화면 전환.

---

## 인기 도서 랭킹 사이드바 (`RankingSidebar`)

- **실시간 인기 도서 TOP 10**: 전체 도서 중 좋아요 수(`like_count`)가 많은 상위 10개 도서를 내림차순 정렬하여 상시 렌더링.
- **독립적 데이터 갱신**: 목록 페이징이나 검색 필터와 관계없이 전체 도서를 기준으로 최신 데이터를 실시간 반영.
- **반응형 모바일 지원**: 화면이 좁은 모바일/태블릿 환경에서는 사이드바가 숨겨지고, 우하단에 플로팅 '도서 랭킹' 버튼이 나타나 클릭 시 모달창 형태로 인기 순위 조회 가능.
- **상세 정보 연동**: 랭킹 항목 클릭 시 즉시 해당 도서의 상세 정보 뷰로 전환.

## 신규 도서 등록 기능 (두 가지 모드)

### 1. API 검색 및 자동 등록 (+ Search)
- **알라딘 도서 검색**: 키워드 입력 시 알라딘 도서 검색 API(Next.js Proxy `/api/aladin` GET)를 호출하여 검색 결과 리스트 출력.
- **최근 검색 기록 저장 및 연동**: 사용자의 검색 키워드를 `search_history` 컬렉션에 자동 저장하고, 최근 검색어 5개를 태그 형태로 제공하여 원클릭 재검색 지원.
- **AI 맞춤 연관 검색어 추천**: OpenAI GPT-4o-mini API(`/api/recommend` POST)를 통해 최근 검색어 5개의 맥락을 분석하여 연관성 높은 3개의 구체적인 추천 키워드를 추출 및 제시.
- **알라딘 상세 지표 활용 자동 평가**: 검색 결과에서 '등록' 버튼 클릭 시, 알라딘 상세 조회 API(`/api/aladin` POST)를 통해 평점(`customerReviewRank`), 판매지수(`salesPoint`), 카테고리, 설명글을 실시간으로 가져옴.
  - **강추 도서 자동 선정**: 평점 8.5점 이상이거나 판매지수 15,000점 이상인 경우 시스템이 자동으로 `bestbook: true` (★ 강추 도서) 설정.
  - **초기 AI 리뷰 작성**: 평가 지표에 따라 자동 추천 문구 또는 일반 소장용 도서 문구를 `ai_review` 필드에 바인딩하여 등록.
- **도서 중복 등록 방지**: `isbn13` 값을 기준으로 기존 PocketBase DB에 동일한 책이 등록되어 있는지 사전 검사(`checkDuplicateIsbn13`)하여 중복 등록 차단.

### 2. 수동 등록 (+ Creator Mode)
- **정보 수동 기입**: 사용자가 제목, 저자, 출판사, 카테고리, 도서 설명, 표지 이미지 URL을 직접 입력하여 등록 가능. (제목, 저자는 필수 입력)
- **인라인 AI 표지 생성기**: 수동 등록 모달 내에 AI 표지 생성 패널을 통합 제공. 도서 제목 및 테마 스타일 선택에 따라 자동으로 묘사 프롬프트를 영문 빌드하여 OpenAI gpt-image-2 API(`/api/genthum`)를 호출하고 이미지를 생성. 적용 및 PC 로컬 저장 가능.

## 도서 상세 정보 및 상호작용 기능 (`BookDetailView`)

- **상세 정보 렌더링**: 도서 썸네일, 제목, 저자, 출판사, 등록일/수정일, 좋아요 수, HTML 형식을 지원하는 상세 책 소개글, AI 리뷰 출력.
- **대출 및 반납 토글 제어**:
  - 도서가 '대출 가능' 상태일 때 로그인 사용자는 '대출하기' 버튼 클릭을 통해 대출 처리 수행 (`isAvailable`을 false로 수정하고 `borrower_id`를 본인 ID로 업데이트).
  - 도서가 '대출 중' 상태일 때, 본인이 대출해 간 사용자(`borrower_id` 일치)인 경우에만 '반납하기' 버튼 활성화. 타인의 도서일 경우 '대출 중 (반납 권한 없음)'으로 버튼 비활성화.
- **도서 삭제**: 본인이 등록한 도서(`user_id` 일치)인 경우에만 상세 정보 뷰에서 '도서 삭제' 버튼 노출. 클릭 시 Confirm 확인 후 DELETE API를 통해 완전 삭제. (대출 상태 변경, 도서 삭제 등의 도서 상태 변경 작업 시 대시보드 차트와 전체 목록이 함께 실시간 갱신됨)

## 도서 좋아요 기능

- **좋아요 설정/해제**: 도서 카드 우하단 하트 아이콘 또는 상세 뷰의 좋아요 버튼 클릭을 통해 활성화.
- **PocketBase likes 컬렉션 연동**: 사용자와 도서 ID의 매핑을 실시간으로 `likes` 컬렉션에 추가/제거하고, 도서 데이터의 `like_count` 값을 자동 증감시킴.
- **실시간 순위 반영**: 좋아요 토글 시 좌측 인기 도서 랭킹 사이드바가 즉각 업데이트됨.

---

## AI thumbnail 표지 이미지 생성 및 파일 저장 기능

- **AI 표지 자동 생성**: 도서 상세 정보 또는 수동 등록 뷰에서 기존 표지가 없거나 생성 기능을 활성화하면 `AiThumbnailGenerator`를 통해 표지 이미지 생성 지원.
- **OpenAI gpt-image-2 연동**: 도서 제목과 저자(수동 등록의 경우 선택한 스타일 테마 및 설명글 추가)를 결합한 영문 프롬프트를 빌드하여 gpt-image-2 모델에 요청 전송. 1024x1536 해상도로 이미지를 생성합니다.
- **서버 측 로컬 파일 시스템 저장**: 생성된 AI 이미지는 프로젝트 백엔드 서버(`/api/genthum`)를 통해 가공되어 프로젝트의 로컬 `public/covers/` 디렉토리에 실시간 파일로 직접 저장됨.
- **상대 경로 DB 맵핑**: PocketBase DB의 `thumbnail` 필드에는 absolute 경로가 아닌 Next.js에서 직접 서빙 가능한 상대 경로(예: `/covers/파일명.png`)로만 저장하여 리소스 경로 정합성 확보.
- **실시간 리렌더링**: 이미지 저장 완료 시 성공 알림과 함께 상세 정보 뷰 및 메인 목록의 썸네일 표지 이미지가 즉시 업데이트됨.

## AI 리뷰 및 요약 스트리밍 기능

- **인라인 AI 요약 및 리뷰 작성**: 도서 상세 뷰 내에 위치한 'AI 리뷰 생성' 버튼 클릭 시 기능 작동.
- **Upstage AI / Solar-pro3 모델 연동**: 브라우저 단에서 직접 Upstage AI API(BaseURL: `https://api.upstage.ai/v1`, Model: `solar-pro3`)에 연결하여 책 제목, 저자, 상세 소개 정보를 제공하고 큐레이팅된 리뷰 생성 요청.
- **실시간 텍스트 스트리밍**: AI가 응답을 생성하는 동안 사용자 화면에 텍스트가 타이핑되듯이 실시간으로 스트리밍하여 출력(`streamedReview` 상태 관리).
- **데이터베이스 반영**: 요약 작성이 완료되면 최종 텍스트 데이터를 PocketBase `books` 컬렉션의 `ai_review` 필드에 PATCH 요청으로 전송하여 저장 및 로컬 캐시 상태 최신화.
- **최신 데이터 강제 동기화**: 상세 정보 뷰 진입 시, 최초 렌더링된 데이터를 덮어쓰지 않고 최신 작성 정보를 유지하도록 PocketBase `books` 단건 조회 API를 강제 호출하여 AI 리뷰 데이터 동기화 수행.
