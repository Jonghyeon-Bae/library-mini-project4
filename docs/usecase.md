# 유스케이스 시나리오

---

## 액터

| 액터       | 설명                  |
| ---------- | --------------------- |
| 사용자     | 웹 브라우저 이용자    |
| Kakao API  | 외부 도서 검색 API    |
| PocketBase | 백엔드 DB 서버 (8090) |

---

## UC-01: 도서 목록 조회

**기본 경로**

1. 사용자가 메인 페이지에 접속한다.
2. PocketBase에서 도서 목록을 조회한다.
3. 도서 카드 그리드가 표시된다.
4. 대출 현황 파이 차트가 함께 표시된다.

**대안 경로**

- **정렬 변경**: "최신순 / 오래된 순 / 제목순" 버튼 클릭 → 해당 기준으로 재정렬
- **도서 0건**: 카드 그리드 비어있으면, 차트 미표시
- **추천 도서**: `bestbook=true`면 뱃지 표시
- **썸네일 없음**: placeholder 이미지로 대체
- **차트 툴팁**: 마우스 오버 시 항목명·수치 표시

**예외 경로**

- **서버 미실행**: 목록 로드 실패

## UC-02: 도서 등록

**기본 경로**

1. 사용자가 "+ 추가" 버튼을 클릭한다.
2. 검색 모달이 열린다.
3. 검색어 입력 후 검색한다.
4. Kakao API에서 결과를 받아 목록에 표시한다.
5. "등록" 클릭 → PocketBase에 저장한다.
6. 알림 표시 후 모달 닫힘, 목록 자동 갱신.

**대안 경로**

- **Enter 검색**: Enter 키로도 검색 가능
- **모달 닫기**: X 버튼 클릭 → 모달 닫힘
- **중복 등록**: 중복 검사 없이 그대로 저장

**예외 경로**

- **빈 검색어**: "검색어를 입력하세요!" alert
- **검색 결과 없음**: 결과 영역 비어있음
- **API 오류**: API Key 만료·네트워크 오류 시 검색 실패

## UC-03: 대출 상태 변경

**기본 경로**

1. 사용자가 "대출 가능" 또는 "대출 중" 버튼을 클릭한다.
2. `isAvailable` 값이 반전되어 PocketBase에 PATCH된다.
3. 목록·차트가 자동 갱신된다.

**대안 경로**

- **토글 반복**: 클릭할 때마다 대출 가능 ↔ 대출 중 전환

**예외 경로**

- **서버 오류**: PATCH 실패 시 UI 변경 없음

---

## UC-04: 도서 삭제

**기본 경로**

1. 사용자가 카드의 "×" 버튼을 클릭한다.
2. confirm 대화상자가 표시된다.
3. "확인" 클릭 → PocketBase에서 삭제된다.
4. 목록·차트가 자동 갱신된다.

**대안 경로**

- **삭제 취소**: "취소" 클릭 → 아무 동작 없음
- **마지막 도서 삭제**: 도서 0건 → 차트 숨김

**예외 경로**

- **삭제 실패**: 서버 오류 시 목록 유지

## UC-05: AI 표지 이미지 생성

**기본 경로**

1. 사용자가 썸네일이 없는 도서에서 이미지 생성 버튼을 클릭한다.
2. 도서의 제목과 내용을 기반으로 GPT Image-2 API에 요청한다.
3. 1024×1536 해상도의 표지 이미지가 생성된다.
4. 생성된 이미지가 PocketBase `thumbnail` 필드에 저장된다.
5. 도서 카드의 썸네일이 자동 갱신된다.

**대안 경로**

- **이미 썸네일 있음**: 기존 썸네일을 새 이미지로 교체
- **재생성**: 결과가 마음에 들지 않으면 다시 생성 가능

**예외 경로**

- **API 오류**: GPT Image-2 통신 실패 시 기존 썸네일 유지
- **내용 부족**: 제목·내용이 비어있으면 생성 불가
