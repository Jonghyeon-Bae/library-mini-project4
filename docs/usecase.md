# 유스케이스 시나리오

---

## 액터

| 액터       | 설명                                                             |
| ---------- | ---------------------------------------------------------------- |
| 사용자     | 웹 브라우저를 통해 도서관 서비스를 이용하는 로그인/비로그인 고객 |
| Aladin API | 외부 도서 데이터 검색 및 평점/판매지수 상세 지표 제공 API       |
| PocketBase | 사용자 계정 관리, 세션 인증 및 도서 CRUD를 수행하는 백엔드 DB    |

| Upstage AI | Solar-pro3 기반 도서 요약 및 리뷰 실시간 스트리밍 생성 API        |

## UC-01: 도서 목록 및 대시보드 조회

**기본 경로**

1. 사용자가 메인 페이지에 접속한다.
2. PocketBase에서 전체 도서 대시보드 통계 데이터 및 1페이지 도서 목록(8개)을 비동기 조회(`useQuery`)한다.
3. 도서 카드 그리드(목록 뷰)와 대출 현황 파이 차트(Recharts)가 화면에 표시된다.
4. 하단 페이지네이션 컨트롤을 이용해 페이지 번호를 클릭하여 다음 도서들을 조회한다.

**대안 경로**

- **정렬 변경**: "최신순 / 오래된 순 / 제목순" 드롭다운 클릭 시 해당 기준으로 정렬 쿼리가 자동 갱신되며 1페이지로 새로고침된다.
- **도서 0건**: 등록된 도서가 없을 경우 대시보드 파이 차트는 렌더링되지 않는다.

## UC-02: 사용자 인증 (회원가입/로그인/로그아웃)

**기본 경로**

1. 비로그인 사용자가 헤더의 '로그인' 또는 '회원가입' 버튼을 클릭한다.
2. 각 모달 창에 필요한 정보(이메일, 패스워드, 회원가입 시 이름 추가)를 입력하고 제출한다.
3. PocketBase Auth API를 호출하여 인증 성공 시 브라우저 쿠키에 세션 상태를 저장한다.
4. 사용자 이름 및 마이페이지 버튼이 헤더에 활성화된다.
5. 로그아웃 버튼 클릭 시 세션 쿠키를 만료시키고 비로그인 상태로 복귀한다.

## UC-03: 도서 검색 및 등록

### 1. API 검색 및 자동 추천 등록
1. 로그인 사용자가 헤더의 "Search!" 버튼을 클릭해 검색 모달을 연다.
2. 키워드를 입력해 도서를 검색하면 알라딘 API(Next.js Proxy)로부터 도서 목록을 받아와 화면에 렌더링하고, 해당 키워드를 최근 검색어(`search_history`)에 저장한다.
3. 검색 결과 중 특정 도서의 '등록' 버튼을 클릭한다.
4. **중복 검사**: `checkDuplicateIsbn13`를 통해 `isbn13`이 기존 DB에 존재하는지 체크한다. (중복 시 알림창 출력 후 중단)
5. **평가지표 조회**: 알라딘 상세 조회 API(POST)를 거쳐 평점 및 판매지수를 조회한다.
   - 평점 8.5 이상 혹은 판매지수 15,000 이상 시 시스템이 자동으로 `bestbook: true` (★ 강추 도서) 설정.
   - 평가지표 수준에 따른 맞춤형 멘트를 `ai_review` 초깃값으로 세팅.
6. DB에 도서 정보가 정상 등록되면 모달을 닫고 전체 도서 목록을 무효화(Invalidate)하여 실시간 동기화한다.

### 2. 수동 등록 (Creator Mode)
1. 로그인 사용자가 헤더의 "CreatorMode!" 버튼을 클릭해 수동 등록 모달을 연다.
2. 제목, 저자, 출판사, 카테고리, 도서 본문 설명글을 직접 입력한다. (제목, 저자는 필수 입력)
3. "등록" 클릭 시 작성 데이터가 PocketBase `books` 컬렉션에 새 레코드로 바로 저장된다.

## UC-04: 도서 상세 조회 및 상호작용 (대출/삭제/좋아요)

**기본 경로**

1. 사용자가 도서 목록 카드 또는 인기 랭킹 항목을 클릭한다.
2. 최상위 화면 상태가 변경되어 목록 뷰가 해당 도서의 상세 정보 뷰(`BookDetailView`)로 전환된다.
3. 사용자가 상세 뷰 하단의 하트 아이콘 또는 '좋아요' 토글을 수행하면 PocketBase `likes` 컬렉션의 상태가 가감되고 책의 좋아요 수(`like_count`)가 반영된다.
4. "목록으로" 버튼을 클릭하여 원래 메인 도서 목록으로 복귀한다.

**대안 경로 (대출 및 반납)**

- **대출하기**: 도서가 '대출 가능' 상태일 때 '대출하기' 클릭 시 `isAvailable`이 false로 변경되고 `borrower_id`가 현재 로그인한 유저의 ID로 업데이트된다.
- **반납하기**: 도서가 '대출 중' 상태이며 대출한 본인(`borrower_id` 일치)인 경우에만 '반납하기' 버튼이 활성화되어 반납 처리를 수행할 수 있다. 타인이 대출한 책은 버튼이 비활성화된다.

**대안 경로 (도서 삭제)**

- 사용자가 본인이 등록한 도서(`user_id` 일치)인 경우 '도서 삭제' 버튼이 노출되며, 클릭 시 Confirm 확인 창을 거쳐 영구 삭제를 요청한다.

## UC-05: AI 표지 이미지 생성

**기본 경로**

1. 사용자가 도서 상세 정보 또는 수동 등록 뷰에서 AI 표지 생성 기능(`AiThumbnailGenerator` 혹은 인라인 패널)을 활성화한다.
2. 수동 등록의 경우 테마 스타일(예: 일러스트, 미니멀, 판타지 등)을 골라 영문 프롬프트를 자동으로 구성할 수 있다.
3. '표지 이미지 생성하기'를 클릭하면 Next.js API `/api/genthum`을 통해 OpenAI gpt-image-2 API를 호출한다.
4. 생성된 이미지가 프로젝트 내 `public/covers/`에 고유 파일로 안전하게 저장된다.
5. DB `thumbnail` 필드에 상대 경로 주소(예: `/covers/filename.png`)가 업데이트되고 화면이 즉시 업데이트된다.

## UC-06: 실시간 인기 도서 랭킹 조회

**기본 경로**

1. 메인 페이지 로드 시 좌측 랭킹 사이드바 컴포넌트(`RankingSidebar`)가 활성화된다.
2. 실시간 쿼리를 통해 전체 도서 중 좋아요(`like_count`) 상위 10개 도서를 찾아 내림차순 리스트를 렌더링한다.
3. 1~3위 항목에는 트로피 아이콘 뱃지가 추가되어 시각화된다.
4. 모바일 환경(가로폭 축소)에서는 좌측 영역이 감춰지고 우하단 플로팅 버튼으로 전환되며, 클릭 시 오버레이 모달 창으로 실시간 랭킹을 보여준다.

## UC-07: AI 리뷰 및 요약 실시간 스트리밍

**기본 경로**

1. 로그인 사용자가 도서 상세 정보 뷰로 진입한다.
2. 'AI 요약' 컴포넌트 우측 상단의 'AI 리뷰 생성' 버튼(Sparkles 아이콘)을 클릭한다.
3. 클라이언트가 Upstage AI API(`https://api.upstage.ai/v1`)에 직접 접속하여 `solar-pro3` 큐레이션 요약 생성을 요청한다.
4. 서버 응답이 단어 단위로 전송되는 동안 프론트엔드가 상태(`streamedReview`)를 실시간 업데이트하여 브라우저 화면에 실시간 타이핑 애니메이션으로 요약 텍스트를 출력한다.
5. 작성이 완료되면 백엔드 PocketBase `books` 컬렉션의 `ai_review` 필드에 패치 업데이트하여 영구 보관한다.

## UC-08: AI 맞춤 검색어 추천

**기본 경로**

1. 로그인 사용자가 헤더의 "Search!" 버튼을 눌러 알라딘 검색 모달을 띄운다.
2. 모달 내에 위치한 'AI 맞춤 연관 검색어' 영역의 '추천어 받기' 버튼을 클릭한다.
3. 최근 검색어 리스트에서 추출한 최대 5개의 검색 키워드를 `/api/recommend` API로 전송한다.
4. OpenAI GPT-4o-mini가 최근 유저의 관심 키워드의 의미와 맥락을 분석하여, 중복이 없는 3개의 연계된 새로운 구체적 추천어들을 JSON으로 응답한다.
5. 모달 화면의 추천어 영역에 3개의 칩 태그가 렌더링되며, 클릭 시 해당 추천어로 즉시 도서 검색이 활성화된다.
