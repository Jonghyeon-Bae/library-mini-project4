# 도서 관리 시스템 요구사항 분석서

## 시스템 개요

- **개인 맞춤형 도서 관리 플랫폼**: 사용자가 도서를 검색 또는 수동으로 등록하고, 대출/반납 상태를 편리하게 관리할 수 있는 웹 기반 플랫폼 제공.
- **AI 융합 도서 서비스**: 생성형 AI 모델(OpenAI, Upstage AI)을 활용하여 AI 맞춤 키워드 추천, 실시간 책 표지(썸네일) 제작, 그리고 실시간 도서 요약/리뷰 스트리밍 서비스를 제공하여 사용자 경험 극대화.

---

## 사용자 목표

- **도서 등록 및 목록 관리**: 알라딘 API를 통한 도서 검색 등록 또는 상세 수동 등록을 통해 서재를 체계적으로 관리.
- **도서 중복 등록 방지**: 중복 검사 기능을 통해 서재 내 도서의 고유성(`isbn13`) 보장.
- **대출 및 반납 제어**: 현재 대출 가능 상태를 실시간 확인하고 본인이 빌린 책에 대한 반납 제어 기능 수행.
- **AI 표지 생성 및 로컬 저장**: 표지가 없거나 마음에 들지 않을 경우 AI 이미지 생성을 통해 표지를 제작하고, 프로젝트 내에 실시간 저장/적용.
- **실시간 AI 도서 요약**: 상세 도서 소개글을 분석해 3~4문장의 핵심 요약 리뷰를 실시간 타이핑 애니메이션(스트리밍) 형태로 즉각 획득.
- **AI 기반 연관 도서 탐색**: 본인의 최근 검색 이력을 기반으로 인공지능이 제안하는 정교한 연관 키워드를 추천받아 신규 도서 탐색 속도 향상.

---

## 데이터베이스 환경 및 설계

- **백엔드 DB 아키텍처**: 데이터베이스 및 사용자 인증 솔루션으로 **PocketBase (Port 8090)** 사용.
- **API 규칙**: 데이터의 입력, 수정, 삭제 및 조회는 PocketBase JS SDK를 사용하여 RESTful API 통신 규칙을 준수함.
- **컬렉션 구조 정책**:
  - `books` 컬렉션: 도서 정보의 메타데이터(제목, 저자, 출판사, 설명, isbn13 등), 대출 상태(`isAvailable`), 대출자 ID(`borrower_id`), AI 리뷰(`ai_review`), 좋아요 수(`like_count`) 관리.
  - `users` 컬렉션: 가입 유저의 계정 정보 및 인증 데이터 관리.
  - `likes` 컬렉션: 유저(`user_id`)와 도서(`book_id`)의 N:M 매핑 데이터 관리.
  - `search_history` 컬렉션: 사용자가 입력한 검색 기록을 일시와 함께 누적 관리.

---

## 데이터 구조 및 저장 정책

- **도서 데이터의 고유 식별**: 모든 등록 도서는 `books` 컬렉션에서 단일 레코드로 고유하게 관리되며, `isbn13` 필드를 통해 중복 데이터 발생 차단.
- **썸네일 이미지 관리 정책**:
  - 데이터베이스의 `thumbnail` 필드는 물리적 바이너리 파일 필드가 아닌 **텍스트/URL 타입** 필드로 사용.
  - 알라딘 검색 등록 시에는 알라딘 측 이미지 URL 주소를 그대로 저장.
  - AI 생성 표지 등록 시에는 서버 로컬 디렉토리(`public/covers/`)에 이미지 파일(`.png`)을 직접 저장한 뒤, DB에는 해당 리소스의 Next.js 상대 경로(예: `/covers/filename.png`)만을 매핑하여 리소스의 독립적 서빙 구조와 정합성 확보.
- **타임스탬프 자동화**: 도서 등록 시 데이터 생성일(`created`) 및 수정일(`updated`)은 PocketBase 서버 측에서 자동으로 타임스탬프를 부여 및 관리.

## 외부 API 연동 명세

시스템은 핵심 비즈니스 로직 및 AI 서비스를 위해 아래와 같이 외부 API를 정교하게 연동함.

### 1. 알라딘 Open API (via Next.js Proxy `/api/aladin`)
- **도서 검색 (GET)**: 사용자가 입력한 검색어를 기준으로 알라딘의 도서 상품 검색 API 호출.
- **평가 지표 및 상세 메타데이터 조회 (POST)**: 도서 고유 번호(`isbn13`)를 이용하여 도서 카테고리 정보, 상세 설명글, 평점 및 판매지수를 확보하여 우수 도서 자동 판별 조건으로 활용.

### 2. OpenAI API
- **AI 썸네일 표지 생성 (`/api/genthum`)**: OpenAI gpt-image-2 모델을 호출하여 도서 제목, 저자, 테마 스타일을 반영한 1024x1536 해상도의 맞춤형 표지 이미지 생성.
- **AI 맞춤 연관 검색어 추천 (`/api/recommend`)**: OpenAI GPT-4o-mini 모델을 호출하여 사용자의 최근 검색어 데이터 5개를 프롬프트 분석하고, 중복되지 않는 새로운 3개의 추천 검색 키워드를 JSON 포맷으로 실시간 반환.

### 3. Upstage AI API (Direct Client SDK Connection)
- **AI 리뷰 및 요약 스트리밍 (`solar-pro3`)**: 상세 페이지 뷰에서 Upstage AI API 엔드포인트(`https://api.upstage.ai/v1`)에 직접 안전하게 접속하여 `solar-pro3` LLM을 호출. 책의 메타 정보를 바탕으로 독자 권장용 리뷰 요약을 생성하고, HTTP Response Stream을 활용해 프론트엔드 UI에 실시간 스트리밍 출력.
